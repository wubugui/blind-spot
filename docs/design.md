# 游戏设计文档

## 📋 项目概述

| 项目 | 内容 |
|------|------|
| **游戏名称** | 盲点 (Blind Spot) |
| **游戏类型** | 点击式视觉推理小说 |
| **核心玩法** | 玩家扮演盲人，在犯罪现场通过对话隐藏身份 |
| **单局时长** | 3-5 分钟 |
| **目标平台** | PC（Windows/Mac/Linux） |
| **技术栈** | HTML + CSS + JavaScript + Electron |

## 🎮 游戏机制

### 怀疑值系统

| 状态 | 数值范围 | 结果 |
|------|----------|------|
| 安全 | 0-59 | 继续游戏 |
| 危险 | 60-79 | NPC 开始警惕 |
| 失败 | ≥ 80 | NPC 识破你，游戏结束 |
| 完美 | 0 | 隐藏结局 |

### 怀疑值变化来源

| 行为 | 变化值 | 说明 |
|------|--------|------|
| 安全对话选项 | 0-5 | 合理回应，不暴露 |
| 可疑对话选项 | 10-25 | 回答不够自然 |
| 危险对话选项 | 25-40 | 明显暴露你知道太多 |
| 点击明显疑点 | 10-20 | 血迹、凶器等 |
| 点击隐蔽疑点 | 5-10 | 细微的不协调 |
| 重复点击同一疑点 | +5/次 | 引起怀疑 |

### 关卡结构

每个关卡包含：

1. **场景信息**
   - 背景图片
   - NPC 信息（头像、名称、描述）
   - 玩家身份（角色、背景故事）

2. **疑点列表**
   - 坐标位置（相对坐标 0-1）
   - 描述文本
   - 怀疑值增量
   - 点击次数限制

3. **对话树**
   - 开场白
   - 5-8 轮对话（每轮 2-4 个选项）
   - 胜利/失败结局台词

## 🛠️ 技术架构

### 模块划分

```
src/js/
├── game.js           # 游戏主循环、入口
├── scene.js          # 场景渲染、热区处理
├── dialogue.js       # 对话树解析、选项处理
├── state.js          # 状态管理（怀疑值、进度）
├── config-loader.js  # 配置加载、热加载
└── utils.js          # 工具函数
```

### 数据流

```
关卡 JSON → config-loader.js → 解析验证
                                      ↓
                              game.js（初始化）
                                      ↓
                    ┌─────────────────┼─────────────────┐
                    ↓                 ↓                 ↓
              scene.js          dialogue.js        state.js
            （渲染场景）        （对话系统）        （状态管理）
                    ↓                 ↓                 ↓
                    └─────────────────┼─────────────────┘
                                      ↓
                              游戏循环进行
```

## 📝 配置规范

详见 [config-spec.md](config-spec.md)

### 核心原则

1. **所有关卡数据外置** - 代码不包含任何硬编码关卡内容
2. **JSON Schema 校验** - 配置必须符合规范
3. **坐标相对化** - 使用 0-1 相对坐标，适配不同分辨率
4. **文本可本地化** - 所有文本可轻松替换

## 🤖 AI 配置支持

### AI 生成流程

1. 用户提供自然语言描述
2. AI 根据提示词生成 JSON 配置
3. 配置校验器检查格式和逻辑
4. 人工审核和调优
5. 放入 `data/levels/` 目录
