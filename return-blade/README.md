# 回刃 RETURN BLADE

一款原创的单键街机小游戏。**整个游戏只有一个操作：点击。**
纯 HTML + Canvas + WebAudio，零依赖，零素材，打开 `index.html` 即玩（支持手机触屏）。

> 本目录与仓库其余内容完全独立，不依赖、不修改任何已有文件。

---

## 一、设计前提：游戏为什么好玩

动手前先拆解了街机/休闲游戏中反复出现的"好玩因子"：

| 因子 | 含义 | 本作的落实 |
|---|---|---|
| **即时反馈** | 每次操作 100ms 内有声光回应 | 打击停顿（hit-stop）、粒子爆裂、屏幕震动、音高随连斩递增 |
| **简单规则 × 涌现深度** | 规则一句话讲完，但局面组合无穷 | 敌人在飞刃飞行期间持续移动，每次出手的时机、角度、深度都是新决策 |
| **风险与回报绑定在同一个动作上** | 危险不是外加惩罚，而是贪心本身的代价 | 掷得越远收割越多，但飞刃离手的空窗期也越长 |
| **失败快、重来快** | 一局 1~3 分钟，死后一键重开 | 即点即重开，无加载、无菜单层级 |
| **可感知的技巧成长** | 玩家变强，而不是数值变强 | 高手技巧全部来自理解规则（见下文"进阶技巧"），数值上没有任何成长系统 |

**核心设计原则：用一条规则同时制造"爽"和"怕"。**

## 二、规则（全部规则就这几行）

- 你是屏幕中央**不能移动**的核心，敌人从四面八方涌来，触核扣 1 格，3 格碎核游戏结束。
- **点击任意位置**：飞刃从核心掷出，飞到点击处后**沿原路返回**。
- 去程与回程都会斩杀沿途敌人。
- **飞刃离手期间，核心毫无防御**；飞刃归位前不能再次出手。
- 计分：单次飞行内第 n 个被斩的敌人得 **n 分**（连斩 5 个 = 1+2+3+4+5 = 15 分）。
  连斩计数在飞刃归位时清零——所以最优解永远是**等敌人排成一串再出手**，而等待本身就是风险。

### 敌人（只有三种，全部服从同一条"冲向核心"规则）

| 敌人 | 特征 |
|---|---|
| 红·猎兵 | 标准速度 |
| 黄·疾袭 | 小而快，难拦截 |
| 紫·重甲 | 慢，需要两刀（一刀去程一刀回程正好处决） |

### 操作

- **点击 / 触摸**：投出飞刃（也是开始 / 重开）
- **P**：暂停 **M**：静音 **R**：（结算画面）重开

## 三、为什么它能"一套规则玩很久"

1. **难度是连续函数**：刷怪间隔与敌速随时间平滑上升，没有关卡边界，每一局都死在自己能力的边缘——这正是心流的形状。
2. **决策密度不衰减**：方向（被包围时救哪边）、深度（掷到敌群前、中、还是穿过头）、时机（再等半秒多串一个，还是现在就出手）三个维度每秒都在重新组合。
3. **计分规则即玩法引导**：n 分递增让"安全的短刀"和"贪婪的长串"产生数量级差距，玩家会自发追逐风险，不需要任何教学。

### 进阶技巧（游戏不会告诉你，但规则会奖励你）

- **穿头刀**：把落点掷到敌群*身后*，去程穿一遍，敌群继续前进时回程再兜一遍。
- **重甲节奏**：对准重甲掷"刚刚过头"的短刀，去程+回程两刀正好处决。
- **空窗管理**：屏幕越乱越要用短刀——长刀的收益要用核心裸奔的时间来换。

## 四、运行

```bash
# 方式一：直接双击打开
return-blade/index.html

# 方式二：本地服务
cd return-blade && npx serve .
```

无构建、无依赖、无网络请求。最高分与最大连斩记录存于浏览器 localStorage。
