## 一句话

你驾驶一艘载人飞艇在一张大陆上旅行,靠在驿站改装你的船,去穿越由**真实大气
物理**构成的极端环境——冰川的下降风、高原的稀薄空气、湖面的推进锋面。

## 三大支柱

1. **风就是关卡**。不摆怪物:下降风急流、湖陆风锋、热泡、山脊风门本身就是
   谜题、陷阱与捷径。核心玩家技能 = 读风。
2. **每一克都算数**。所有改装在"质量 ↔ 浮力 ↔ 功率"铁三角里交易
   (真实飞艇工程,引擎原生行为,不造假数值)。
3. **备航与应变**。驿站里按天气情报做准备;空中用操作和应急手段
   (抛压舱/放氦/借热泡/迎风锚定)处置意外。

## 核心循环

```
驿站:看地图与天气 → 机库改装(试航评级) → 接货/补给 → 选航线与出发时机
  ↓
航段飞行(5~10 分钟实况,平静段可加速):读风、选高度带、管理电量/浮力
  ↓
抵达驿站:结算货物与耗材 → 赚钱 → 解锁更远区域 → 循环(可回头重走)
```

## "不硬核"的具体设计(松紧标尺)

- **失败温和**:坠毁 = 救援队送回上一驿站,船自动修好,损失本次货物 + 一笔
  修理费。**永不删档、永不清空仓库**。重试成本 ≤ 一段航程(几分钟)。
- **辅助默认开**:高度保持/航向保持(= 引擎现有 PID)从第一艘船就配好;
  手动精细操作是加分项不是门槛。定点悬停/自动迎风是可购买的进阶航电。
- **天气情报基本可靠**:预报给"风向 + 强度档位 + 时段",小幅随机偏差;
  不搞恶意 RNG。赌的空间来自"要不要等到明早再出发"这类主动选择。
- **改装看仪表不看公式**:机库"试航评级"后台跑真仿真,直接给
  净浮力/静稳定/最大可顶风速三个表盘 + 绿黄红评级。
- **难度靠"门槛"不靠"惩罚"**:高原不改大气囊就是飞不进去(物理如此),
  但一旦装备合格,准备充分的航段成功率应 ≈ 90%。
- 目标体验节奏:顺利时看风景,紧张时手心出汗,失败时"下次带对装备再来"。

## 载人级参数集(量级推导,实现时进 config 注释)

- 囊体 25m × 8m 长椭球:V ≈ 838 m³,海平面排开空气 ≈ 1026 kg,
  氦(含杂质 0.18 kg/m³)≈ 151 kg → **毛浮力 ≈ 875 kg**。
- 蒙皮 ≈ 513 m² × 250 g/m² ≈ 128 kg;结构+吊舱+系统 ≈ 250 kg;
  飞行员 80 kg;动力+电池 ≈ 150 kg → **载荷/压舱/改装预算 ≈ 250 kg**。
  这 250 kg 就是铁三角的"钱包"。
- 巡航 10 m/s 轴向阻力 ≈ 230 N → 双桨各 150 N(可升级 300 N);
  功率 ≈ 4 kW,20 kWh 电池(≈100 kg)≈ 5 小时巡航——数值天然成立。
- 世界距离做戏剧化压缩:一段航程仿真内 3~6 km、巡航 10~15 m/s,
  实况 5~8 分钟(平静段允许 ×2~5 时间加速,引擎已支持)。

## 大陆结构(长存档)

- **节点图大陆**:4~5 个区域、15~20 个驿站,驿站间 1~3 条备选航线。
  可自由来回;已走过的航线随天气/昼夜变化仍有重玩价值。
- **区域按环境难度渐进解锁**(靠物理门槛,不靠钥匙):
  1. **绿原**(教学):温和边界层,学操艇/读风/停泊
  2. **湖区**:湖陆风锋时刻表、水上无迫降
  3. **山地**:山脊风门、背风回流、峡谷
  4. **高原**(大门槛):低密度,必须大气囊+大推力改装
  5. **冰川**(终区):下降风急流走廊、日落骤重、终点城
- **经济**:货运合同(货有质量!接大单牺牲机动)、区域差价、发现奖励
  (风景点/沉船/气象站)。氦气是最贵耗材(缓慢泄漏 + 主动排气消耗)。
- **昼夜与日历**:热泡白天生效、坡风昼夜反向、日落氦冷缩变重——
  "在驿站过夜等清晨"是正经策略。
- **存档**:localStorage 自动存 + JSON 导出/导入。确定性种子 → 天气可复盘。

## 环境 → 关卡模板(全部已有物理)

| 生物群系 | 主导机制 | 玩法 |
| --- | --- | --- |
| 绿原 | 对数边界层 | 教学;安全迫降随处可 |
| 湖区 | 湖陆风锋(时变)、水面无热泡 | 算锋面时刻表;水上迫降=丢货 |
| 山地 | 山脊加速、背风回流 | 风门冲越;背风陷阱 |
| 高原 | 低密度(浮力/推力打折)、白天热泡、超温 | 改装门槛;热泡电梯省电 |
| 冰川谷 | Prandtl 下降风急流(高度分层)、反照 | 顺风=贴地高速走廊;逆风=爬出急流层 |
| 沙漠(后加) | 强热泡、大昼夜温差 | 白天颠簸有免费升力,夜里骤重 |

关键一致性:**地形与风场同源生成**——冰川谷的坡向 = katabatic 下坡向,
山脊的走向 = 风门方向(航段规格里成对定义,不允许对不上)。

## 改装系统(机库)

| 槽位 | 梯度示例 | 代价(铁三角) |
| --- | --- | --- |
| 气囊 | 838 → 1200 → 1800 m³ | 浮力↑ = 侧风受风面↑ + 蒙皮重↑ + 氦耗↑ |
| 动力 | 150N 巡航桨 → 300N 强风桨 → 矢量桨 | 质量↑ 耗电↑,高原推力仍 ∝ρ |
| 尾翼 | 小翼 → 大十字翼 → 可调舵面 | 静稳定↑ = 质量+阻力↑ |
| 能源 | 电池组数、太阳能蒙皮 | 电池重;太阳能昼夜差 |
| 压舱/氦 | 可抛压舱袋、氦气瓶、排气阀 | 应急资源,用了就没 |
| 涂层 | 银(超温小)↔ 深(白天浮力增益) | 昼夜双刃 |
| 航电 | 高度保持→航向保持→定点/自动迎风 | 引擎 PID 模块化商品 |
| 货舱 | 容量与减震等级 | 货 = 质量 |

**试航评级**(机库核心功能):改装确认前后台跑无头仿真,输出三表盘——
净浮力(按目的地海拔/温度)、静稳定裕度(尾翼 vs Munk)、最大可顶风速
(自动扫描)。绿黄红一目了然,物理后果所见即所得。

## 航段飞行(moment-to-moment)

- **HUD**:风矢量(当前+目的方向)、净浮力表、电量、高度带风廓线小图
  (现有垂直剖面图改造)、到站距离。
- **操作**:WASD+QE 手动(现有)+ 辅助开关;应急热键:抛压舱、放氦、
  全速迎风锚定、请求救援。
- **到站**:驿站上空停泊圈内低速保持数秒 = 系留成功(用现有定点逻辑判定)。
- **航段内事件**(轻量):阵风锋(预报会提示时段)、氦阀小泄漏(提示音+
  缓慢变重,撑到驿站即可)、顺风走廊/热泡链(正向惊喜,省时省电)。

## 技术架构

**铁律:工程仿真保持纯净。** 游戏只 import `airship_sim`;新增物理进引擎时
仍按工程标准(`[假设]` 注释 + 单元测试),游戏逻辑一律不进引擎。

```
airship-sim/
  airship_sim/            ← 引擎(现有,继续当工程工具用)
    ground.py             ← 新:地形高度场 + 接触/碰撞(可选模块,None 时零改动)
    energy.py             ← 新:电池/功率(P≈k·T^1.5/√ρ 动量理论)/太阳能(可选模块)
    presets_crewed.py     ← 新:载人级参数集(带推导注释)
  game/
    DESIGN.md             ← 本文档
    index.html            ← 游戏页(复用 Pyodide 加载器 + Three.js 壳)
    game.py (内嵌)        ← 游戏态:大陆图/经济/存档/航段编排(跑在 Pyodide)
    legs/                 ← 航段规格(地形+风场+目标成对定义,JSON)
```

- 改装只发生在驿站(停靠态)→ 换 config 后**干净重建** Simulation,
  无飞行中换物理的状态延续问题。
- 地形:每航段由种子确定性生成的解析高度场(谷/脊/坡),与风场参数同源。
- 碰撞 MVP:艇底/吊舱采样点对地形的弹簧-阻尼接触;垂直触地速度 >3 m/s
  或撞向陡坡 = 坠毁(救援流程)。驿站停泊圈内接地 = 合法系留。
- 部署:与现有页面同套 GitHub Pages,游戏在 `/airship-sim/game/`。

## 实现笔记(开发中发现的真实问题)

1. **大惯量船不吃航向阶跃**:90° 偏航设定跳变 → 偏航环饱和 → 侧滑增大 →
   Munk 力矩(∝u·v)超过全部差速权限 → 甩艏(broach)螺旋。解法:航向设定
   回转限速(0.12 rad/s)+ **转弯协调**(航向误差大时自动收油,先转正再提速)
   ——这就是真实飞艇的操纵规程。
2. **载人尺度尾翼重定标**:25m 船 S=8m² 尾翼在 6 m/s 侧风下航向发散;
   S=30m²+电机横距 4m 后漂移 <2m。目录里的"大型尾翼 48m²"达到完全静稳定。
3. **放氦的正确物理**:排出氦的体积由 ballonet 进气充填(囊体积不变),
   质量净增 (ρ_air−ρ_he)·V_vented——第一版"只减氦质量"方向反了,测试抓住。
4. **爬升/下降航段的浮力学**:按高端海拔配平 → 低端偏轻,靠浮力免费爬升,
   到顶近中性;下坡途中放氦增重下降。这是真实浮空器运行方式,也是游戏教学点。
5. **性能**:游戏物理用 5ms RK4(载人艇最高频≈接触 2Hz,精度损失可忽略),
   采样率保持整数 tick(IMU=2/控制=4/气压计=20),CPython 5.3× 实时,
   Pyodide 留足余量。
6. **地形保持辅助**(默认开):沿航向前视 12s 航程,把高度设定顶到地形+28m,
   只升不降——"不过于硬核"的落地形式;可关闭(坠机测试即关闭它复现撞山)。
