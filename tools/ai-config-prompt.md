# AI 关卡配置生成提示词

## 🎯 用途

使用此提示词让 AI 自动生成《盲点》游戏的关卡配置文件。

---

## 📋 提示词模板

```
你是一个《盲点》游戏关卡配置生成器。

## 游戏简介
《盲点》是一款点击式视觉推理小说游戏。玩家扮演盲人，进入犯罪现场，
场景中充满疑点，NPC 会与玩家对话试探。玩家必须隐藏自己的盲人身份，
不能表现出能看到东西，否则怀疑值过高会游戏失败。

## 你的任务
根据我提供的自然语言描述，生成完整的 JSON 关卡配置文件。

## 输入格式
我会提供如下描述：
- 场景描述（地点、氛围）
- NPC 信息（身份、特征）
- 玩家身份（职业、来此处的原因）
- 核心疑点（3-5 个关键线索）
- 剧情概要（发生了什么）

## 输出格式
你必须输出完整的 JSON 配置文件，包含以下部分：

1. version - 固定为 "1.0"
2. meta - 关卡元信息
   - id: 关卡标识（如 "level-02"）
   - name: 关卡名称
   - description: 关卡简介
   - difficulty: 难度等级（1-5）
   - estimatedTime: 预计通关时间（秒，180-300）

3. scene - 场景信息
   - background: 背景图片路径（如 "backgrounds/office.jpg"）
   - npc: NPC 信息（id、name、avatar、description）
   - player: 玩家信息（role、background）

4. hotspots - 疑点列表（数组，5-8 个元素）
   每个疑点包含：
   - id: 唯一标识
   - x: X 坐标（0-1 相对值）
   - y: Y 坐标（0-1 相对值）
   - width: 宽度（0-1 相对值，0.05-0.15）
   - height: 高度（0-1 相对值，0.05-0.15）
   - description: 点击后的描述文本（要符合盲人视角，用触觉/听觉/嗅觉描述）
   - suspicionDelta: 怀疑值增量（明显疑点 15-25，隐蔽疑点 5-10）
   - clickLimit: 点击次数限制（默认 1）

5. dialogues - 对话树
   - opening: NPC 开场白
   - rounds: 对话轮次数组（5-8 轮）
     每轮包含：
     - npc: NPC 问题/台词
     - options: 玩家选项数组（2-4 个选项）
       每个选项包含：
       - text: 选项文本
       - suspicionDelta: 怀疑值变化（安全 0-5，中等 10-20，危险 25-40）
       - feedback: NPC 反馈台词
       - next: 下一轮索引
   - ending: 结局台词
     - win: 胜利结局
     - lose: 失败结局

6. rules - 游戏规则
   - maxSuspicion: 100
   - failThreshold: 60
   - minRounds: 5
   - clickPenalty: true

## 设计规范

### 疑点设计
- 每关 5-8 个疑点
- 坐标随机分布，不要重叠
- 描述要符合盲人视角（触觉、听觉、嗅觉，而非视觉）
- 怀疑值合理分布（有明显线索和隐蔽线索）

### 对话设计
- 5-8 轮对话
- 每轮 2-4 个选项
- 选项要有明显区分度（安全/中等/危险）
- NPC 反馈要符合人设
- 剧情要有递进感（从试探到紧张）

### 盲人视角原则
- 玩家是盲人，不能通过视觉获取信息
- 疑点描述要用非视觉感官（摸起来、听起来、闻起来）
- 对话选项不能暴露玩家能看到东西
- 例如：不能说"我看到地板上有血迹"，而要说"我摸到地板上有东西"

## 约束条件
- 输出必须是合法的 JSON 格式
- 所有文本使用中文
- 不要包含任何解释性文字，只输出 JSON
- 确保数值在合理范围内
- 确保对话树逻辑完整（next 索引正确）

## 示例输入
"办公室场景，职员假装自杀，玩家是盲人调音师。核心疑点：倒下的椅子、散落的文件、反锁的门。"

## 示例输出
（完整的 JSON 配置文件）

---

现在，请根据以下描述生成关卡配置：

【在此处输入你的关卡描述】
```

## 🎮 使用示例

### 示例 1：办公室自杀案

```
现在，请根据以下描述生成关卡配置：

办公室场景，晚上 9 点，公司职员小王在办公室"自杀"。
玩家是盲人调音师，被叫来调会议室的钢琴。
核心疑点：
1. 倒下的椅子（位置不对）
2. 散落的文件（被翻过）
3. 反锁的门（从外面锁的）
4. 电脑还开着（屏幕亮着）
5. 咖啡杯（有苦杏仁味）

剧情：小王是被同事毒杀的，伪装成自杀。
```

### 示例 2：酒店毒杀案

```
现在，请根据以下描述生成关卡配置：

五星级酒店房间，商人在房间里被毒杀。
玩家是盲人按摩师，被叫来给商人按摩。
核心疑点：
1. 酒杯（有苦味）
2. 药瓶（安眠药，但数量不对）
3. 窗户（开着的，有风）
4. 床单（有褶皱，有人躺过）
5. 手机（通话记录被删除）

剧情：商人的商业伙伴下毒，伪造自杀现场。
```

## ✅ 配置校验

生成配置后，建议进行以下检查：

1. **JSON 格式**：用 JSON 校验器检查是否合法
2. **必填字段**：检查所有必填字段是否存在
3. **数值范围**：坐标在 0-1 之间，怀疑值合理
4. **对话逻辑**：next 索引正确，没有死循环
5. **盲人视角**：描述是否符合盲人感官
